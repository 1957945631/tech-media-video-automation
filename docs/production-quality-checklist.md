# 成片生产质检清单

这份清单用于后半段生产：五日筛选和口播确认后，有最终配音和 SRT，Codex 负责找素材、生成 visual beats、渲染成片并审查质量。

## 生产边界

后半段必须从以下文件开始：

- `data/video-scripts/YYYY-MM-DD-voiceover.md`
- `data/audio/YYYY-MM-DD-voiceover.mp3`
- `public/audio/YYYY-MM-DD-voiceover.mp3`
- `data/subtitles/YYYY-MM-DD-aligned.srt`

后半段不得再用口播文本估算字幕时间。口播文本只用于理解语义、提取关键词和规划素材。

## 渲染前检查

必须运行：

```bash
npm test
npm run validate:timeline
```

通过标准：

- SRT 可解析，无倒序、无重叠。
- `durationSeconds` 来自最终音频真实时长。
- `captions` 来自最终 SRT。
- `segments` 不越界。
- `visualBeats` 不越界、不跨热点、不和 `captionRange` 时间冲突。
- 所有 visual beat 至少有一个素材。
- 观众版数据不得包含 `risk`、`审核提醒` 或内部制作提示。
- 每个 visual beat 必须包含 `assetFunction`，用于驱动组件库选择。
- `remotion_motion_clip` 和 `yellow_opinion_card` 属于组件渲染素材，可以没有外部 asset 文件；其他素材函数仍必须有可读取素材路径。
- fallback 卡不得叠加全局来源角标、关键词 chips 或重复正文。

## 素材检查

每个 visual beat 检查：

- 画面语义是否对应当前口播句。
- 是否填写 `subject`、`action`、`concept`、`visualRole` 和 `transitionOut`。
- `visualRole` 是否服务当前句的语义功能。
- 真实网页截图是否可读。
- 自制信息卡是否只是补充解释或节奏点，而不是替代真实素材成为默认画面。
- 不用无意义占位图糊弄。
- 不能让口播讲 A，画面已经跳到 B。
- `evidence` 截图必须能看出来源和核心事实，但不要求烙入红框。
- 固定坐标红框、固定箭头、未确认位置的放大镜禁止使用。
- 需要标注时，只能使用人工确认、语义安全的宽区域标注、裁切、下划线或局部放大。
- `keyword` 必须是短判断句，不能只是普通描述。
- 真实 b-roll 优先使用视频素材；缺视频时可以用真实图片做动态推拉兜底。

优先级：

1. 官方来源截图、产品页、发布页和演示页。
2. 可信新闻源截图。
3. 监管、交易所、论文、GitHub、开发者文档截图。
4. 语义贴合口播的可追溯视频素材。
5. 语义贴合口播的可追溯图片素材。
6. Remotion 语义动画片段，用于机制、流程、依赖关系、风险扩散等真实素材不足的抽象内容。
7. 自制语义信息卡，仅用于解释、总结和节奏补充。

## 字幕检查

字幕必须符合：

- 无背景板。
- 大号白字。
- 黑色描边或投影。
- 居中偏下。
- 和配音对齐。
- 不遮挡素材主体。
- 断句自然，不能一行过长导致溢出。

## 视觉检查

必须避免这些低级错误：

- 中文乱码。
- 文字互相遮挡。
- 外层黄字、标签、标题盖住素材正文。
- 字幕背景板误用。
- 字幕和素材主体重叠。
- 装饰元素压住新闻网页标题或正文。
- 红框、箭头、放大镜指向空白区域或和网页内容错位。
- 画面切换早于或晚于当前口播主题。
- 素材长时间空白、黑屏、加载失败。
- 重复使用和当前句无关的素材。
- 同一 URL 或同一域名被过量重复分配。
- fallback 卡冒充真实证据截图。
- Remotion motion clip 统计口径和最终 episode 不一致。
- 同一种 `visualRole` 连续超过 2 次。
- 每 15 秒没有真实世界素材。
- 每 20 秒没有结构拆解、证据、产品 UI 或关键判断支撑。
- 用抽象科技画面凑数，或者把内部素材类型名显示给观众。
- 任何 10 秒内没有信息变化。
- 观众画面出现“审核提醒”、素材缺口、fallback/generated 状态或内部制作文案。

素材上方可以有来源标签，但来源标签不能盖住页面标题。外层标题和关键词标签如果影响素材可读性，优先删除。

## 抽帧审查

至少抽查：

- 开头 10 秒。
- 每个热点切换点前后。
- 每个 visual beat 的中点，或每个热点至少 2 个中点。
- 字幕密集处。
- 结尾 20 秒。

人工判断：

- 当前字幕是否和配音一致。
- 当前素材是否和口播句一致。
- 素材正文是否能看清。
- 画面是否有低级遮挡。
- 整体审美是否统一。

## 渲染后检查

必须运行：

```bash
npm run validate:timeline:video
```

通过标准：

- 视频真实时长约等于音频时长。
- SRT、segments、visualBeats 均未超过视频时长。
- 音频存在且可播放。
- 输出视频路径正确。

## 完成标准

只有同时满足以下条件，才算完成：

- 自动校验全部通过。
- 完整视频已重新渲染。
- 抽帧审查没有发现遮挡、乱码、错位。
- 人工快速看过开头、每个热点开头和结尾。
- 成片质量问题已修复并重新验证。

如果渲染成功但审查发现问题，不能交付。必须修复后重新渲染。
