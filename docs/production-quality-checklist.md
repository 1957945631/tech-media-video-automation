# 成片生产质检清单

这份清单用于后半段生产：你提供最终配音和 SRT 后，Codex 负责找素材、生成 visual beats、渲染成片并审查质量。

## 生产边界

前半段自动化任务只负责新闻、口播和分镜草稿。

后半段必须从以下文件开始：

- `data/audio/YYYY-MM-DD-voiceover.mp3`
- `public/audio/YYYY-MM-DD-voiceover.mp3`
- `data/subtitles/YYYY-MM-DD-aligned.srt`

后半段不得再用口播文本估算字幕时间。口播文本只用于理解语义、提取关键词和规划素材。

## 渲染前检查

必须运行：

```bash
npm test
npm run validate:timeline
```

通过标准：

- SRT 可解析，无倒序、无重叠。
- `durationSeconds` 来自最终音频真实时长。
- `captions` 来自最终 SRT。
- `segments` 不越界。
- `visualBeats` 不越界、不跨热点、不和 captionRange 时间冲突。
- 所有 visual beat 至少有一个素材。

## 素材检查

每个 visual beat 检查：

- 画面语义是否对应当前口播句。
- 真实网页截图是否可读。
- 自制信息卡是否统一风格、信息清晰。
- 不用无意义占位图糊弄。
- 不能让口播讲 A，画面已经跳到 B。

优先级：

1. 官方来源截图。
2. 可信新闻源截图。
3. 监管、交易所、论文、GitHub、开发者文档截图。
4. 自制语义信息卡。

## 字幕检查

字幕必须符合：

- 无背景板。
- 大号白字。
- 黑色描边或投影。
- 居中偏下。
- 和配音对齐。
- 不遮挡素材主体。
- 断句自然，不能一行过长导致溢出。

## 视觉检查

必须避免这些低级错误：

- 中文乱码。
- 文字互相遮挡。
- 外层黄字、标签、标题盖住素材正文。
- 字幕背景板误用。
- 字幕和素材主体重叠。
- 装饰元素压住新闻网页标题或正文。
- 画面切换早于或晚于当前口播主题。
- 素材长时间空白、黑屏、加载失败。
- 重复使用和当前句无关的素材。

素材上方可以有来源标签，但来源标签不能盖住页面标题。外层标题和关键词标签如果影响素材可读性，优先删除。

## 抽帧审查

至少抽查：

- 开头 10 秒。
- OpenAI 段 3 个时间点。
- 每个热点切换点前后。
- 每个 visual beat 的中点，或每个热点至少 2 个中点。
- 字幕密集处。
- 结尾 20 秒。

人工判断：

- 当前字幕是否和配音一致。
- 当前素材是否和口播句一致。
- 素材正文是否能看清。
- 画面是否有低级遮挡。
- 整体审美是否统一。

## 渲染后检查

必须运行：

```bash
npm run validate:timeline:video
```

通过标准：

- 视频真实时长约等于音频时长。
- SRT、segments、visualBeats 均未超过视频时长。
- 音频存在且可播放。
- 输出视频路径正确。

## 完成标准

只有同时满足以下条件，才算完成：

- 自动校验全部通过。
- 完整视频已重新渲染。
- 抽帧审查没有发现遮挡、乱码、错位。
- 人工快速看过开头、每个热点开头和结尾。
- 成片质量问题已修复并重新验证。

如果渲染成功但审查发现问题，不能交付。必须修复后重新渲染。
