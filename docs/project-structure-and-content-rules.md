# 项目结构与内容规范

## 频道定位

《一周科技大事》面向普通科技新闻观众，不是内部汇报，也不是单一 AI 周报。内容应从一周科技事件里筛出值得讲清楚的变化，兼顾行业重要性、普通人可理解性和画面表达能力。

AI 可以出现，但应和芯片、消费电子、互联网平台、云计算、开源安全、机器人、新能源、电力、航天、游戏、硬件等方向共同构成科技大事池。

## 分层职责

- `src/types/`：稳定类型，不放当期生成数据。
- `src/data/`：当前可渲染 episode 数据，由构建脚本生成。
- `src/components/`：Remotion 视觉组件库。
- `src/rendering/`：时间线、素材选择、调度逻辑。
- `scripts/pipeline/`：生产流程脚本。
- `scripts/`：兼容入口、工具函数和测试。
- `config/`：新闻、筛选、视觉系统规则。
- `docs/`：流程说明和质检规范。

## 观众画面禁区

以下内容不得进入 Remotion 成片画面：

- `审核提醒`
- `risk`
- 内部素材缺口提示
- “生成失败”“fallback”等制作提示
- 给创作者看的流程说明

这些信息只能出现在日志、报告或内部 JSON 里。

## 素材组合

最佳组合是：真实证据 + 产品界面 + 产业画面 + 结构拆解 + 黄色观点卡。

素材数量建议：

- 证据截图：8-15 张
- 产品 UI：5-10 张
- 真实 b-roll：20-35 段或张
- 结构拆解：5-10 个
- 黄色观点卡：6-10 个

`annotation` 不再作为库存目标。证据截图默认不烙入固定红框、固定箭头或固定坐标标注。

## 节奏规则

- 同一视觉角色不能连续超过 2 次。
- 同一主体再次出现时必须换表现方式。
- 每 15 秒至少出现一次真实世界素材。
- 每 20 秒至少出现一次图解或观点卡。
- Logo 不能撑满超过 3 秒。
- 不用“抽象科技画面”凑数；缺少贴合真实素材时，改成观众能看懂的信息卡或结构拆解。
- 抽象画面之后必须接现实素材或证据素材。

证据截图可以使用来源标签、裁切、宽区域强调、下划线或局部放大，但必须是语义安全的。禁止固定坐标红框、固定箭头和无法确认位置的标注。

## 执行边界

本项目的后半段以最终 MP3 和 SRT 为唯一时间基准。口播草稿只帮助理解语义，不决定最终字幕和画面时长。

渲染成功不等于成片合格。每次正式成片前必须通过自动校验，并进行人工抽帧检查。
